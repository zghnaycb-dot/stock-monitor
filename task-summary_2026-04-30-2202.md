# Task Summary: Fix UnicodeEncodeError in Stock Monitor App

## Objective
Fix `UnicodeEncodeError: 'utf-8' codec can't encode characters: surrogates not allowed` that prevented Streamlit app from starting.

## Root Cause
Multiple Python source files (`alerts.py`, `data_source.py`, `enhanced_source.py`, `kline_chart.py`, `stock_compare.py`, `backtest.py`) were written with `\uXXXX` unicode escape sequences instead of actual UTF-8 characters. When Python tried to render these strings (especially emoji like `\ud83d\udd14` for 🔔), the surrogate pairs were invalid for UTF-8 encoding.

This happened because `_build_modules.py` wrote unicode escapes into the source files instead of raw UTF-8 characters.

## Fixes Applied

### 1. alerts.py (was empty after failed fix)
- File was 0 bytes after previous failed `unicode_escape` decoding attempt
- Reconstructed from `.pyc` bytecode analysis + manual rewrite
- Replaced emoji with ASCII text (`[ALERT] Price Alert`) to avoid encoding issues

### 2. Unicode Escape Decoding (5 files)
Used a custom decoder (`_fix_unicode4.py`) that:
- Detects `\uXXXX` escape sequences
- Properly combines surrogate pairs (e.g., `\ud83d` + `\udd14` → 🔔)
- Rewrites files with pure UTF-8 encoding
